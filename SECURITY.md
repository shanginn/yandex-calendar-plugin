# Политика безопасности

## Поддерживаемые версии

Исправления безопасности выпускаются для последней версии ветки `0.1.x`.

## Как сообщить об уязвимости

Не создавайте публичную GitHub Issue с секретами или готовым эксплойтом. Напишите Nikolai Shangin на <shanginn@gmail.com> с темой `Yandex Calendar plugin security`.

Укажите версию, способ воспроизведения, возможное влияние и минимальный пример без реальных паролей и персональных данных. Получение отчёта будет подтверждено по возможности быстро; срок исправления зависит от серьёзности и воспроизводимости проблемы.

## Модель угроз MVP

- пароль приложения поступает только через окружение процесса;
- сервер не записывает календарные данные и секреты на диск;
- запросы уходят только на origin настроенного CalDAV-сервера;
- по умолчанию разрешён только HTTPS;
- результаты не содержат заголовок Authorization и пароль приложения;
- удаление требует отдельного подтверждения.

Компрометация локального компьютера, процесса Codex или самого аккаунта Яндекса находится вне модели угроз. При подозрении на утечку немедленно отзовите пароль приложения в Яндекс ID.
