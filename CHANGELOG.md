# История изменений

Все заметные изменения проекта документируются в этом файле.

## 0.1.3 — 2026-07-17

- добавлен destructive MCP-инструмент `delete_calendar` для нативного удаления коллекции через CalDAV `DELETE`;
- перед удалением обязательны свежий `list_calendars`, точное совпадение `calendar_url` и `confirm=true`;
- URL повторно разрешается по актуальному серверному списку, а результат возвращает имя удалённого календаря;
- основной календарь определяется по `schedule-default-calendar-URL`; основной и последний календари защищены, а при невозможности определить default операция отменяется;
- добавлены unit- и MCP-интеграционные тесты неверного URL, отсутствующего подтверждения, уже удалённого календаря, основного/последнего календаря и успешного удаления;
- добавлен opt-in live-тест, который создаёт через `MKCALENDAR` и удаляет только уникальный временный календарь.

## 0.1.2 — 2026-07-16

- `list_events` теперь строго фильтрует пересечение с запрошенным полуоткрытым диапазоном;
- повторяющиеся события разворачиваются локально с учётом `RRULE`, `RDATE` и `EXDATE`;
- `RECURRENCE-ID` overrides, отменённые экземпляры и `RANGE=THISANDFUTURE` применяются к occurrence;
- orphan overrides вне диапазона больше не попадают в ответ из-за ошибочного совпадения на стороне CalDAV;
- добавлены регрессионные unit- и MCP-интеграционные тесты для бага со встречами 2023 года.

## 0.1.1 — 2026-07-16

- добавлен приватный файл учётных данных `~/.config/yandex-calendar-plugin/credentials.json` для Codex Desktop;
- файл с секретом принимается только с правами `0600` на Unix;
- переменные окружения сохранены как приоритетный способ настройки и могут переопределять файл;
- добавлены тесты чтения и проверки прав файла.

## 0.1.0 — 2026-07-16

Первый production-ready MVP:

- пять MCP-инструментов для чтения и изменения Яндекс Календаря;
- CalDAV discovery, calendar-query, ETag и безопасные PUT/DELETE;
- ссылка Яндекс Телемоста для создаваемых событий;
- автономная локальная сборка и публичный Codex marketplace;
- русская документация, privacy policy, terms и submission checklist;
- Apache-2.0 с обязательным сохранением NOTICE автора;
- typecheck, форматирование, unit- и end-to-end интеграционные тесты.
