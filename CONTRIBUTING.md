# Участие в разработке

Спасибо за интерес к проекту.

1. Создайте fork и отдельную ветку.
2. Не добавляйте реальные логины, пароли приложений и календарные данные.
3. Выполните `npm install` и `npm run check`.
4. Опишите изменение, риски и проверки в pull request.
5. Сохраните `LICENSE`, `NOTICE` и copyright автора в производных работах.

Предложения по расширению MVP сначала лучше обсудить в GitHub Issues. Интеграционные тесты должны использовать только локальный поддельный CalDAV-сервер.
